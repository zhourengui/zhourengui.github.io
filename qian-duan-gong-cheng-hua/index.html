前端工程化入门

前言：

前端工程化 是指将前端开发的流程规范化、标准化，包括开发流程、技术选型、代码规范、构建发布等，用于提升前端工程师的开发效率和代码质量。

为什么要前端工程化：

  • 复杂度高：前端项目的多功能、多页面、多状态、多系统
  • 规模大：团队开发，多人协作，代码质量管理
  • 要求高：页面性能优化（CDN/异步加载/请求合并），CSS兼容性、单页面应用、服务端渲染。。。

怎么做前端工程化：

  • 从业务着手：简单的单页面应用，使用gulp打包+同步工具实现开发全流程。
  • 从复杂度考虑：jenkenis、git/gitlab、webpack、React/Vue/Angular

Webpack

  • 入口：
module.exports = {
    entry: ".."
}
  • 输出：
const path = require("path")
module.exports = {
    output: {
        path: path.resolve(__dirname, "dist"),
        filename: "webpack.bundle.js"
    }
}
  • loaders：处理静态文件
module.exports = {
    module: {
        rules: [
            { test: /\.txt$/, use:  'raw-loader' }
        ]
    }
}
  • plugins: 插件
const HtmlWepackPlugin = require("html-webpack-plugin")
module.exports = {
    plugins: [ new HtmlWepackPlugin({ template: ".." })  ]
}
  • 模式/兼容性

yeoman创建自定义脚手架

安装yeoman：npm install -g yo
安装generator-generator：npm install -g generator-generator

然后在glup项目中执行yo generator

将我们之前写好的demo复制到generator-glup-zhourengui>genertors>app>templates

配置generator-glup-zhourengui>genertors>app>index.js：

writing() {
    this.fs.copy(
      this.templatePath("**"), // 代表将templates的文件复制到generator
      this.destinationPath("./")
    );
  }

测试：然后在generator-glup-zhourengui执行npm link，就会这个目录下的npm的依赖包连接到全局，

然后在根目录下创建一个测试文件夹，在这个测试文件夹执行yo glup-zhourengui，执行的时候可能会报bower不存在，然后配置generator-glup-zhourengui>genertors>app>index.js：

iinstall() {
    this.installDependencies({ bower: false });
}

将脚手架发布到npm：

npm publish

使用yeoman总结：

  • 脚手架的名称是generator-打头
  • 全局安装generator-generator
  • link命令本地测试，发布时设置npm registry
