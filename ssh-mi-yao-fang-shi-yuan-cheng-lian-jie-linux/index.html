SSH密钥方式远程连接Linux

前言：

我们经常使用账号密码的方式登陆服务器，这种方式还是有点繁琐，我们可以通过ssh密钥的方式远程连接Linux。

修改服务器ssh默认端口

服务器的ssh默认端口一般是22，也可以用过netstat -anlp | grep sshd命令查看默认的端口，如下：

可以看到我的LInux服务器的sshd默认端口是10022

接下来我们就来修改sshd的默认端口：

同vi /etc/ssh/sshd_config编辑文件，默认有个# Port 22，我们可以把井号去掉，然后修改需要的端口，这里我是用的是10022端口。

然后AuthorizedKeysFile这个几点默认是.ssh/authorized_keys的，不建议修改，如果修改了的小伙伴，在下面的讲述中将authorized_keys修改成自己修改的文件即可。

然后就是一定我们自定义添加的端口了，使用semanage port -a -t ssh_port_t -p tcp 10022，如果没有semanage命令的话请查找下资料，自行安装，运行了这段代码后，我们可以验证我们这个命令是否操作成功，使用semanage port -l | grep ssh，如下图：

说明了我们之前的代码运行成功。

最后重启sshd，service sshd restart。

ssh密钥登陆

首先我们需要在本地的.ssh目录下通过ssh-keygen创建密钥，我创建的密钥名是baiduyun_key，.ssh目录下就会生成baiduyun_key和baiduyun_key.pub文件。

创建完之后，修改服务器上的/.ssh/authorized_keys文件，将上一步生成的baiduyun_key.pub的密钥复制到authorized_keys文件中。

最后一步就是修改本地的/.ssh/config文件，文件中添加

Host littleboy
  Port 10022
  HostName 服务器ip地址
  User root
  IdentityFile ~/.ssh/baiduyun_key
  IdentitiesOnly yes

到这里就所有的工作完成了，可以通过ssh littleboy验证是否设置成功。
